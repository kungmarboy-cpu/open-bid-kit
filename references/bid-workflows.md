# OpenBidKit_Yibiao 标书制作提示词参考

本文件整理自 OpenBidKit_Yibiao（易标）项目中的核心提示词模板，
供 Codex agent 在标书制作各环节中直接使用。

---

## 1. 标书分析任务模板

### 系统提示（通用）

```
你是专业的投标资料分析助手。请严格基于用户提供的上下文完成提取和总结。

通用要求：
1. 保持信息全面、准确，优先使用用户提供上下文中的内容；除非具体任务明确要求或允许根据经验补充，否则不要自行编造
2. 如果上下文没有提及，明确写"没有提及"
3. 只输出最终结果，不输出过程、提示语或客套话
4. 始终使用简体中文
```

### 任务列表

| ID | 任务 | 必须 | 输出 | 说明 |
|----|------|------|------|------|
| projectOverview | 项目概述 | 是 | markdown | 项目基本信息、背景目的、规模预算、时间安排、实施内容和技术特点 |

## 2. JSON任务模板

用于结构化数据提取：

```
任务：{任务名称}

目标：{提取目标}

约束：
1. 输出格式必须为 JSON。
2. 严格按照以下 JSON 格式输出，只修改 value，禁止修改 key 和结构。
3. 招标文件中没有的字段填充"没有提及"。

JSON 格式：
{JSON结构示例}

仅输出 JSON，不要输出其他内容。
```

## 3. 技术评分要求分类提取

提取时区分"技术评分项"和"技术评分要求"：

- 技术评分项：需要在技术方案中一一响应、展开编写的具体评分内容（方案类、措施类、团队类、实施类等）
- 技术评分要求：约束评分、解释评分、定义扣分规则的通用说明（符合性要求、偏离扣分规则、适用范围等）

---

## 4. 目录生成规则

一级目录对应技术评分大类，每个一级目录下包含：

- 二级目录：归纳后的二级主题
- 三级目录：响应要点、实施措施、证明材料或验收标准

编号规则：
- 一级: 1, 2, 3...
- 二级: 1.1, 1.2...
- 三级: 1.1.1, 1.1.2...

## 5. 废标项检查

### 分析阶段

梳理"无效投标"和"废标项"中哪些能通过电子投标文件内容判断。
排除签字、盖章、密封、纸质正副本、现场递交等线下事项。
结合投标文件目录和正文结构，指出重点核查章节。

### 检查阶段

输出格式要求：
- 检查项描述
- 风险等级（高/中/低）
- 检查依据
- 检查结果（通过/不通过/需确认）
- 处理建议

---

## 6. 内容生成规范

- 响应对应的评分标准和要求
- 结合项目特点出具具体实施方案
- 专业、准确，符合行业规范
- 包含实施方法、步骤、工具、交付物、保障措施
- 使用格式规范：标题层级、段落和列表
- 在适当位置插入架构图、流程图、进度表
